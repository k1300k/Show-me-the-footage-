# 📱 모바일 뷰 테스트 가이드

브라우저에서 모바일 화면을 확인하는 방법입니다.

## 🖥️ Chrome/Edge에서 모바일 뷰 보기

### 방법 1: 개발자 도구 사용 (추천)

1. **브라우저 열기**
   ```
   http://localhost:3000
   ```

2. **개발자 도구 열기**
   - Windows/Linux: `F12` 또는 `Ctrl + Shift + I`
   - Mac: `Cmd + Option + I`

3. **디바이스 툴바 토글**
   - Windows/Linux: `Ctrl + Shift + M`
   - Mac: `Cmd + Shift + M`
   - 또는 개발자 도구 좌측 상단 📱 아이콘 클릭

4. **디바이스 선택**
   - 상단 드롭다운에서 선택:
     - `iPhone 14 Pro` (393 x 852)
     - `iPhone SE` (375 x 667)
     - `Samsung Galaxy S20` (360 x 800)
     - `iPad Air` (820 x 1180)
   - 또는 `Responsive`로 설정 후 폭을 `375px`로 조정

5. **확인할 사항**
   - ✅ 컴팩트한 헤더 ("🎥 CCTV" + LIVE 배지)
   - ✅ 상단 고정 검색창
   - ✅ 가로 스크롤 해시태그
   - ✅ 2열 CCTV 그리드
   - ✅ 하단 네비게이션 바 (홈/지도/즐겨찾기)

## 🍎 Safari에서 모바일 뷰 보기

1. **브라우저 열기**
   ```
   http://localhost:3000
   ```

2. **개발자 도구 활성화** (처음 한 번만)
   - Safari > 환경설정 > 고급
   - "메뉴 막대에서 개발자용 메뉴 보기" 체크

3. **반응형 디자인 모드**
   - 메뉴: 개발 > 반응형 디자인 모드 시작
   - 단축키: `Cmd + Option + R`

4. **디바이스 선택**
   - iPhone 14 Pro
   - iPhone SE
   - iPad

## 🧪 테스트 시나리오

### 1. 검색 테스트
```
1. 검색창에 "강남역" 입력
2. Enter 또는 전송 버튼 클릭
3. 2열 그리드에 결과 표시 확인
4. CCTV 썸네일 클릭
5. 하단 시트 열림 확인
```

### 2. 해시태그 테스트
```
1. 상단 해시태그를 좌우로 스크롤
2. "#강남역" 클릭
3. 검색 결과 즉시 표시 확인
```

### 3. 네비게이션 테스트
```
1. 하단 "지도" 버튼 클릭
2. 지도 뷰로 전환 확인
3. "홈" 버튼 클릭
4. 목록 뷰로 복귀 확인
5. "즐겨찾기" 버튼 클릭
6. 즐겨찾기 페이지 이동 확인
```

### 4. CCTV 영상 테스트
```
1. CCTV 썸네일 클릭
2. 하단 시트 85vh 높이 확인
3. 📹 버튼으로 영상/썸네일 전환
4. ⭐ 버튼으로 즐겨찾기 추가
5. 시트 외부 터치로 닫기
```

### 모바일 (<768px)
- 컴팩트 헤더
- 하단 네비게이션 바

- 가로 스크롤 해시태그

### 데스크톱 (≥1024px)

- 상단 토글 버튼

- 2줄 해시태그 래핑
- 600px 고정 시트

✅ **스크롤 숨김**: 가로 스크롤 바 제거

## 💡 실제 모바일 기기 테스트

### 같은 네트워크에서 접속

1. **서버 URL 확인**
   ```
   터미널에서 확인:
   - Network: http://192.168.X.X:3000
   ```

2. **모바일 기기에서 접속**
   - 같은 Wi-Fi 연결
   - 브라우저에서 Network URL 입력

3. **홈 화면에 추가**
   - iOS: Safari 공유 메뉴
   - Android: Chrome 메뉴
